# <a name="optional-diagnostic-data-for-office"></a>Необов’язкові діагностичні дані для Office

> [!IMPORTANT]
> Інформація в цій статті стосується версії 1904 або новіших версій переліченого нижче клієнтського програмного забезпечення Office, інстальованого на комп’ютері з Windows.
> - Програми Microsoft 365 для підприємств (раніше мали назву Office 365 ProPlus)
> - Програми Microsoft 365 для бізнесу (раніше мали назву Office 365 Business)
> - Microsoft 365 Персональний, Microsoft 365 Сімейний або інші версії Office, які входять до передплати на Microsoft 365.
> - Класичні програми Project і Visio, які постачаються з певними планами передплати, як-от Project (план 5) або Visio (план 2).
>
> Ця інформація також стосується версії 16.28 або пізнішої версії перелічених нижче програм Office для Mac: Excel, Outlook, OneNote, PowerPoint і Word.

Діагностичні дані служать для підтримки безпеки та актуальності Office, виявлення, діагностування та усунення проблем, а також для вдосконалення продукту. До цих даних не входить ім’я користувача або адреса електронної пошти, вміст файлів, а також відомості про програми, не пов’язані з Office.

Ці діагностичні дані, які збираються й надсилаються до корпорації Майкрософт, описують клієнтське програмне забезпечення Office, що використовується на комп'ютерах із Windows. Деякі діагностичні дані обов’язкові, а деякі – ні. За допомогою елементів керування конфіденційністю, зокрема параметрів політики для організацій, ви маєте змогу вибрати, обов’язкові чи необов’язкові дані будуть надсилатися. Ви можете переглянути діагностичні дані, що надсилаються, за допомогою засобу перегляду діагностичних даних.

***Необов’язкові діагностичні дані*** – це додаткові дані, які допомагають нам удосконалювати продукт, а також надають розширені відомості для виявлення, діагностики та усунення проблем.

Необов’язкові діагностичні дані збираються разом з обов’язковими.

До необов’язкових діагностичних даних належать, наприклад, дані про фігури, які користувачі вставляють у документи Word, які збираються для покращення пропозицій, а також дані про швидкість відображення слайдів PowerPoint на екрані, які збираються, щоб прискорити показ слайдів.

Щоб більше дізнатися про діагностичні дані, зверніться до таких статей:

## <a name="categories-of-optional-diagnostic-data"></a>Категорії необов’язкових діагностичних даних

Необов’язкові діагностичні дані поділяються на такі категорії:

- Налаштування та оновлення ПЗ
- Використання продуктів і служб
- Швидкодія та безперебійність продуктів і служб
- Підключення та конфігурація пристрою

Ці категорії відображаються в засобі перегляду діагностичних даних і мають такі самі категорії, що й обов’язкові діагностичні дані.

У наступних розділах наведено опис кожної категорії та приклади подій для кожної категорії.

## <a name="software-setup-and-inventory-events"></a>Події налаштування та оновлення ПЗ

Ця категорія містить події, які можуть охоплювати такі області:

- Назва та версія інстальованого продукту, стан інсталяції.
- Програмні надбудови та їхні параметри.
- Помилки документів, функцій та надбудов, які можуть зашкодити безпеці, зокрема вплинути на стан готовності до оновлення продукту.

У таблиці нижче наведено приклади подій у цій категорії та опис цих подій.

| **Назва події**   | **Опис події**  |
| ---- | ---- |
| Office.Extensibility.AppCommands.GetRibbonUpdatesForUserId | Ця подія вказує, чи успішно відбувається оновлення стрічки в інтерфейсі користувача Word, коли користувач заходить в інший обліковий запис. Ми використовуємо цю подію для виявлення неправильного налаштування та інших проблем, які можуть вплинути на інтерфейс користувача Office. |
| Office.Extensibility.AppCommands.AppCmdInstall   | Ця подія містить відомості про надбудову Office, інстальовану користувачем, зокрема ідентифікатор програми, збірку та версію операційної системи, успішність і тривалість інсталяції.  |

## <a name="product-and-service-usage-events"></a>Події використання продуктів і служб

Ця категорія містить події, які можуть охоплювати такі області:

- Успішність функціонування програми. Обмежується операціями відкривання та закривання програм і документів, редагування файлів і спільного доступу до файлів (спільної роботи).
- Визначення, чи сталися події конкретних функцій, таких як запуск або припинення, а також чи працює ця функція.
- Функції спеціальних можливостей Office

У таблиці нижче наведено приклади подій у цій категорії та опис цих подій.

| **Назва події**   | **Опис події**  |
| ------ | ------- |
| Office.Word.Commanding.Highlight  | Ця подія вказує, що у Word було виконано команду виділення тексту. Ми використовуємо цю подію для виявлення помилок у команді виділення тексту.  |
| Office.Translator.AddInLoaded   | Підтвердження того, чи функцію перекладача успішно завантажено й відображено інтерфейс.  |
| Office.Graphics.GVizInsertShape |Відстежує використання функції вставлення фігур у Word, а також указує, фігури якого типу вставлено та з якого джерела.| 
| Office.PowerPoint.PPT.Desktop.SummaryZoomInsertionRule   | Ця подія визначає, чи присутні в документі будь-які розділи, коли користувач вставляє інтерактивний зміст, а також чи видаляє користувач наявні розділи. |
| Office.Security.SecureReaderHost.ProtectedViewValidation | Відстежує, коли та з якою метою відкривають файл у безпечному поданні. Використовується для виявлення ситуацій, коли безпечне подання не активується належним чином, щоб забезпечити правильну роботу функції. |

## <a name="product-and-service-performance-events"></a>Події швидкодії та безперебійності продуктів і служб

Ця категорія містить події, які можуть охоплювати такі області:

- Неочікуване закриття (аварійне завершення роботи) програми та стан програми на цей момент.
- Повільна або погана робота, коли програма запускається чи відкривається файл.
- Помилки функціоналу та взаємодії з користувачем.

У таблиці нижче наведено приклади подій у цій категорії та опис цих подій.

| **Назва події**    | **Опис події**   |
| --------------- | -------------- |
| Office.Word.Word.CoreSaveTime100ns     | Ця подія реєструє швидкість збереження документа програмою Word. Ми використовуємо цю подію, щоб виявляти помилки та проблеми зі швидкодією під час збереження документів у Word.|
| Office.Identity.SignInForWamAccountAad  | Ця подія фіксується, коли користувач входить до облікового запису Azure Active Directory через бібліотеку диспетчера облікових записів в Інтернеті (WAM). Ця подія надсилає метадані, як-от AppName, AppVersion і код помилки в разі невдачі. |
| Office.PowerPoint.PPT.Desktop.FileOpen.FirstSlideMasterThumbnailRenderTime | Ця подія повідомляє про загальний час, необхідний для виводу на екран першого ескізу зразка слайдів у PowerPoint.  |
| Office.Extensibility.Diagnostics   | Ця подія надає загальні діагностичні відомості про надбудови Office, як-от звіти про аварійне завершення роботи для потреб налагодження.|
